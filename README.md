# boatrace-ev-notify

ボートレースの期待値(EV)買い目を毎日自動でDiscordに通知するシステム。
サーバー不要・GitHub Actionsのみで自走します。

## 通知の流れ

| タイミング | 内容 |
|---|---|
| 毎朝 8:00 頃 (JST) | 全開催場を走査し **暫定EV TOP10**(前売オッズ基準・参考値) |
| 各レース締切 約8分前 | 最新オッズで再計算した **確定版EV**(EV≥1.2のみ、最大3点) |

EV = モデル予測確率 × オッズ。**EV≥1.2のみ通知、1.1未満は厳禁**(表示もしない)。

## セットアップ(15分)

1. **Discord Webhook作成**: 通知したいチャンネル → 設定 → 連携サービス → Webhook作成 → URLコピー
2. このフォルダを **publicリポジトリ** としてGitHubへpush
   - privateだとActions無料枠(月2,000分)を直前ループが数日で使い切ります。publicは無制限
3. リポジトリの Settings → Secrets and variables → Actions →
   `DISCORD_WEBHOOK_URL` にWebhook URLを登録
4. Actionsタブ → 各workflowの `Run workflow`(手動実行)で動作確認

## ⚠️ 初回に必ずやること: パーサー検証

HTMLの構造は変わりうるため、パース部分はローカルで一度検証してください:

```bash
pip install -r requirements.txt
python -m src.scraper   # 本日の開催場・出走表・オッズの取得テスト
```

出走表6艇分の `win_rate` / `motor_rate` / `st_avg` と、オッズ120通りが
正しく取れているかを boatrace.jp の実ページと突き合わせて確認。
ズレていたら `src/scraper.py` の正規表現/セレクタを調整します。

## 設計方針

- **確率モデル** (`src/model.py`): コース別基礎勝率 × 選手勝率 × モーター2連率 × 平均ST
  を Plackett-Luce で3連単確率に展開。まずは統計ベースで開始し、
  的中ログが溜まったらLightGBM等に差し替え可能な構造
- **理由の自動生成**: 市場想定確率との乖離、モーター上位、ST優位、
  1号艇過剰人気(A1ブランド売れすぎ)などの既知バイアスをテキスト化
- **負荷配慮**: 1リクエスト1秒以上のスリープ
- **通知疲れ防止**: 直前通知はEV≥1.2のレースのみ。該当なしなら沈黙

## 制約・注意

- 朝8時時点の前売オッズは薄く、締切までに大きく変動します(暫定版はあくまで参考)
- GitHub Actionsのcronは数分〜数十分遅延することがあります
- 控除率約25%のため、EV>1の買い目が毎日あるとは限りません。
  「通知ゼロの日」が正常な日も多い想定です
- boatrace.jpのHTML構造変更でパースが壊れた場合、Discordに警告が飛びます
